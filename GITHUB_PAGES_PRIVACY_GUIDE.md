# GitHub Pages Privacy Policy Publishing Guide

## Recommended publishing method

Push the repository to:
- `https://github.com/starhunt/threads-clipper-for-obsidian`

Then enable **GitHub Pages** for the repository.

### Simplest setup
1. Push the repo
2. Open repository **Settings**
3. Go to **Pages**
4. Under **Build and deployment**:
   - **Source**: `Deploy from a branch`
   - **Branch**: `main`
   - **Folder**: `/ (root)`
5. Save

## Expected public URL

After Pages is enabled, the privacy policy URL will typically be:

- `https://starhunt.github.io/threads-clipper-for-obsidian/privacy-policy.html`

If your GitHub username or repo name changes, update the URL accordingly.

## What to submit to Chrome Web Store

Use the GitHub Pages URL above in the **Privacy Policy** field.

## Why this is better than a GitHub blob URL

Prefer Pages URL over:
- `https://github.com/.../blob/main/PRIVACY_POLICY.md`

Because Pages gives you:
- a clean public web page
- no GitHub file viewer chrome
- a more official-looking policy page for review

## Optional next step

If you want, you can also add a small link from `README.md` to:
- `privacy-policy.html`

so the public repository and the published policy stay easy to find together.
