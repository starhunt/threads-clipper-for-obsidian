## 1. 전용 목적
좋아요 또는 저장한 Threads 게시글을 사용자의 로컬 Obsidian 앱에 노트로 빠르게 보관하는 확장 프로그램입니다. 사용자가 Threads에서 게시글을 명시적으로 선택했을 때만 동작하며, 게시글 본문, 링크, 작성자 정보, 날짜 등의 정보를 정리된 Markdown 형태로 만들어 Obsidian URI를 통해 사용자의 볼트에 새 노트로 생성합니다.

---

## 2. storage 사용 근거
사용자 설정과 로컬 사용 통계를 저장하기 위해 storage 권한이 필요합니다. 예를 들어 좋아요/저장 트리거 활성화 여부, 볼트 이름, 저장 폴더, 연/월 하위 폴더 사용 여부, 마지막 활성 볼트 저장 여부 같은 설정을 저장합니다. 또한 총 저장 개수와 오늘 저장 개수 같은 간단한 로컬 통계도 저장합니다. 이 데이터는 확장 프로그램 기능 제공 목적에만 사용됩니다.

---

## 3. activeTab 사용 근거
사용자가 현재 보고 있는 Threads 페이지의 게시글을 기준으로 저장 동작을 수행하기 위해 activeTab 권한이 필요합니다. 이 확장 프로그램은 사용자가 Threads 게시글에서 좋아요 또는 저장 버튼을 눌렀을 때만 현재 탭의 콘텐츠를 읽고, 해당 게시글의 본문/작성자/링크/날짜/미디어 링크를 추출합니다. 다른 목적의 탭 접근에는 사용하지 않습니다.

---

## 4. notifications 사용 근거
게시글이 Obsidian으로 전달되었는지 사용자에게 즉시 알려주기 위해 notifications 권한이 필요합니다. 저장이 준비되면 사용자는 알림을 통해 성공 여부를 확인할 수 있고, 이 알림은 확장 프로그램의 핵심 기능인 저장 피드백 제공 목적에만 사용됩니다. 광고, 마케팅, 리마인더 용도로 사용하지 않습니다.

---

## 5. clipboardWrite 사용 근거
이 출시판은 Obsidian Local REST API가 아니라 Obsidian URI(`obsidian://new`) 방식을 사용합니다. 따라서 새 노트를 만들 때 노트 본문을 클립보드를 통해 Obsidian으로 전달해야 하므로 clipboardWrite 권한이 필요합니다. 사용자가 좋아요 또는 저장 동작으로 게시글 저장을 명시적으로 트리거했을 때에만 클립보드에 노트 내용을 기록합니다.

---

## 6. 호스트 권한 사용 근거
확장 프로그램은 Threads 게시글을 저장하기 위해 Threads 웹사이트에서만 동작해야 하므로 `https://www.threads.net/*` 및 `https://www.threads.com/*` 호스트 권한이 필요합니다. 또한 게시글에 포함된 이미지/미디어 링크를 읽어 노트 안에 함께 기록하기 위해 Threads가 사용하는 미디어 CDN 호스트 권한이 필요합니다. 이 호스트 권한은 사용자가 저장을 트리거한 게시글의 정보 추출과 미디어 링크 기록 목적에만 사용됩니다.

---

## 7. 원격 코드 사용 근거
원격 코드를 사용하지 않습니다. 모든 실행 코드는 확장 프로그램 패키지 내부에 포함되어 있으며, 외부 서버에서 JavaScript, Wasm 또는 기타 실행 코드를 내려받아 실행하지 않습니다. 이 출시판은 AI 서비스, 원격 스크립트 로더, eval 기반 코드 실행, 외부 코드 삽입 기능을 포함하지 않습니다.
